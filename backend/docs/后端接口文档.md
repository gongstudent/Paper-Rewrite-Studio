# 后端接口文档

- 文档版本：`v1.0`
- 编写日期：`2026-03-21`
- 接口前缀：`/api`

## 1. 统一规范

## 1.1 通用响应结构

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "requestId": "req_001"
}
```

失败示例：

```json
{
  "code": 40001,
  "message": "模型连接失败",
  "data": null,
  "requestId": "req_002"
}
```

## 1.2 统一请求头

- `Content-Type: application/json`
- `Authorization: Bearer <token>`，一期可选保留

## 1.3 任务状态枚举

- `pending`
- `running`
- `done`
- `failed`

## 1.4 通用错误码建议

| code | 含义 |
| --- | --- |
| `40001` | 参数错误 |
| `40002` | 文件格式不支持 |
| `40003` | 文档不存在 |
| `40004` | 模型配置不存在 |
| `40005` | 任务不存在 |
| `50001` | 文件解析失败 |
| `50002` | 模型调用失败 |
| `50003` | 模型连接失败 |
| `50004` | 导出失败 |

## 2. 工作台接口

## 2.1 获取首页概览

### `GET /api/dashboard/overview`

返回：

```json

  "paperCount": 24,

  "todayRewriteAcceptRate": 86
}
```

{
  "items": [

      "docId": "doc_001",
      "title": "基于大语言模型的学术写作伦理研究",

## 3. 文档接口

## 3.1 上传论文

### `POST /api/documents/upload`

请求类型：`multipart/form-data`

字段：

- `file`
- `title`
- `language`

返回：

```json
{
  "docId": "doc_001",
  "title": "人工智能辅助写作对学术规范影响研究",
  "sourceFile": {
    "name": "draft.docx",
    "type": "docx"
  },
  "version": 1
}
```

## 3.2 获取论文列表

### `GET /api/documents`

返回：

```json
{
  "items": [
    {
      "docId": "doc_001",
      "title": "人工智能辅助写作对学术规范影响研究",
      "language": "zh-CN",
      "version": 1,
      "updatedAt": "2026-03-21T09:00:00+08:00"
    }
  ]
}
```

## 3.3 获取论文详情

### `GET /api/documents/{docId}`

返回：

```json
{
  "docId": "doc_001",
  "title": "人工智能辅助写作对学术规范影响研究",
  "language": "zh-CN",
  "version": 3,
  "sections": [],
  "paragraphs": [],
  "citations": []
}
```

## 3.4 重新解析论文

### `POST /api/documents/{docId}/parse`

请求：

```json
{
  "excludeCover": true,
  "excludeCatalog": true,
  "excludeReferences": true,
  "excludeAppendix": false
}
```

返回：

```json
{
  "taskId": "parse_001",
  "status": "pending"
}
```

## 3.5 保存排除范围

### `PATCH /api/documents/{docId}/exclusions`

请求：

```json
{
  "excludeCover": true,
  "excludeCatalog": true,
  "excludeReferences": true,
  "excludeAppendix": false,
  "manualExcludedSegmentIds": ["seg_11", "seg_12"]
}
```

## 4. 模型配置接口

## 4.1 获取模型列表

### `GET /api/model-providers`

## 4.2 新增模型配置

### `POST /api/model-providers`

请求：

```json
{
  "providerType": "local",
  "name": "本地 Ollama",
  "baseUrl": "http://127.0.0.1:11434",
  "apiKey": "",
  "model": "qwen2.5:7b",
  "capabilities": ["rewrite", "detect"],
  "timeoutMs": 60000,
  "concurrency": 2,
  "contextWindow": 8192
}
```

## 4.3 更新模型配置

### `PUT /api/model-providers/{providerId}`

## 4.4 删除模型配置

### `DELETE /api/model-providers/{providerId}`

## 4.5 测试模型连接

### `POST /api/model-providers/{providerId}/test`

返回：

```json
{
  "status": "ok",
  "latencyMs": 853,
  "message": "连接成功"
}
```

## 4.6 设置默认模型

### `POST /api/model-providers/{providerId}/set-default`

请求：

```json
{
  "scene": "rewrite"
}
```

`scene` 可选值：

- `rewrite`
- `detect`

## 6. 改写接口

## 6.1 发起改写任务

### `POST /api/rewrite-tasks`

请求：

```json
{
  "docId": "doc_001",
  "segmentIds": ["seg_1", "seg_2"],
  "strategy": "lower_aigc",
  "providerId": "provider_001",
  "model": "qwen2.5:7b",
  "options": {
    "tone": "academic",
    "preserveTerms": true,
    "preserveCitations": true,
    "preserveNumbers": true
  }
}
```

## 6.2 获取改写任务结果

### `GET /api/rewrite-tasks/{taskId}`

返回：

```json
{
  "taskId": "rw_001",
  "status": "done",
  "progress": 100,
  "candidates": [
    {
      "candidateId": "cand_001",
      "segmentId": "seg_001",
      "rewrittenText": "改写后的内容",
      "explanation": "保留术语与数字，对句式进行重组",
      "beforeScore": 82,
      "afterScore": 46,
      "accepted": false
    }
  ]
}
```

## 6.3 采纳候选版本

### `POST /api/rewrite-candidates/{candidateId}/accept`

返回：

```json
{
  "candidateId": "cand_001",
  "accepted": true
}
```

## 6.4 回退候选版本

### `POST /api/rewrite-candidates/{candidateId}/rollback`

## 6.5 再次改写

### `POST /api/rewrite-tasks/{taskId}/retry`

## 8. 导出接口

## 8.1 创建导出任务

### `POST /api/exports`

请求：

```json
{
  "docId": "doc_001",
  "exportType": "diff_report"
}
```

`exportType` 可选值：

- `final_doc`
- `diff_report`
- `summary_report`

返回：

```json
{
  "exportId": "exp_001",
  "status": "pending"
}
```

## 8.2 获取导出结果

### `GET /api/exports/{exportId}`

返回：

```json
{
  "exportId": "exp_001",
  "status": "done",
  "downloadUrl": "/downloads/exp_001.docx"
}
```

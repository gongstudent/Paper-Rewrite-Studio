# 后端需求文档

- 文档版本：`v1.0`
- 编写日期：`2026-03-21`
- 适用对象：`后端开发 / 架构 / 接口负责人`

## 1. 目标

后端负责承载论文降重工具的一期业务能力，为前端提供统一、稳定、可轮询的 API 和任务系统。

后端核心目标：

- 支持论文文件上传、解析与结构化
- 支持模型配置、模型测试与模型调用
- 支持检测、改写、复检、导出等异步任务
- 提供统一的数据对象与接口响应规范

## 2. 后端模块划分

## 2.1 工作台与总览模块

职责：

- 统计论文数量
- 汇总高风险段数量
- 返回最近论文
- 返回最近任务
- 返回当前活跃任务

## 2.2 文档管理模块

职责：

- 上传原始文件
- 保存文档元数据
- 解析正文结构
- 保存章节、段落、引用信息
- 保存版本快照
- 维护排除范围设置

## 2.3 模型配置模块

职责：

- 管理本地模型和云端模型配置
- 持久化默认模型
- 执行模型连通性测试
- 统一生成模型调用配置

## 2.4 检测任务模块

职责：

- 创建检测任务
- 执行重复风险检测
- 执行 AIGC 风险检测
- 保存段落级结果
- 输出总览结果

## 2.5 改写任务模块

职责：

- 根据段落和策略创建改写任务
- 调用模型生成改写候选
- 保存候选结果
- 支持采纳、回退、重试

## 2.6 复检任务模块

职责：

- 对已采纳段落重新检测
- 输出前后分值变化
- 更新文档处理状态

## 2.7 导出任务模块

职责：

- 生成最终稿
- 生成带差异标记稿
- 生成检测摘要报告

## 4. 任务处理要求

## 4.1 异步任务范围

以下操作建议必须异步：

- 文档解析
- 检测
- 改写
- 复检
- 导出

## 4.2 状态定义

所有任务统一使用：

- `pending`
- `running`
- `done`
- `failed`

可选扩展：

- `cancelled`

## 4.3 结果保留要求

- 任务失败时保留错误信息
- 已完成段落不可因单段失败而全部丢失
- 改写候选需要保留历史版本

## 5. 模型调用要求

后端需要统一支持两类模型来源：

- 本地模型：`Ollama`、`OpenAI-compatible 本地网关`
- 云端模型：`OpenAI-compatible` 抽象适配

模型调用适配层至少应支持：

- 文本生成
- 连通性测试
- 超时控制
- 错误码统一

## 6. 文件处理要求

- 支持 `doc`、`docx`、`pdf`、`txt`
- 文件上传后需生成唯一存储路径
- 解析结果需要结构化到章节和段落级别
- 用户排除的段落不能参与检测与改写

## 7. API 设计要求

- 使用 REST 风格
- 统一响应结构
- 统一错误码
- 列表接口支持分页扩展
- 导出接口支持下载地址或文件流

## 8. 后端非功能要求

### 8.1 稳定性

- 模型调用失败不应影响已有文档数据
- 任务异常需要记录日志和 requestId

### 8.2 安全性

- API Key 默认脱敏存储和返回
- 上传文件需要有存储清理策略
- 预留 `Authorization` 头支持后续扩展

### 8.3 可扩展性

- 后续可扩展第三方检测报告导入
- 后续可扩展多模型路由
- 后续可扩展多用户和权限体系

## 9. 后端验收标准

- 能完成文档上传、解析、检测、改写、复检、导出闭环
- 任务状态可被前端稳定轮询
- 模型配置可保存、测试和切换默认值
- 检测结果能定位到段落级
- 改写结果能支持候选版本与采纳回退

## 10. 后端默认假设

- 一期使用单体后端即可，不强制拆微服务
- 文件可先存本地磁盘或对象存储抽象层
- 一期默认单用户，不实现复杂登录权限

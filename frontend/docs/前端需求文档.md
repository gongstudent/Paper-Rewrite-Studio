# 前端需求文档

- 文档版本：`v1.0`
- 编写日期：`2026-03-21`
- 适用对象：`前端开发 / 前端负责人 / 产品`

## 1. 目标

前端负责实现论文降重工具的一期 Web 页面、交互流程和状态反馈，确保用户可以顺畅完成上传、检测、改写、复检和导出。

前端实现重点不在复杂动画，而在于：

- 信息层级清晰
- 高风险内容定位清晰
- 原文与改写对比清晰
- 长耗时任务反馈清晰

## 2. 页面与路由建议

| 页面 | 建议路由 | 说明 |
| --- | --- | --- |
| 工作台首页 | `/workspace` | 总览、最近论文、最近任务、快捷操作 |
| 论文详情页 | `/documents/:docId` | 文档解析结果、排除范围、版本信息 |
| 检测报告页 | `/reports/:taskId` | 风险总览、正文高亮、段落详情 |
| 改写对比台 | `/rewrite/:docId` | 原文、改写稿、采纳、回退、进度 |
| 模型配置页 | `/models` | 本地模型与云端模型配置 |

建议补充弹窗级路由或状态：

- 上传论文弹窗
- 导出弹窗
- 错误提示弹窗
- 删除模型确认弹窗

## 3. 全局前端要求

### 3.1 顶层布局

- 左侧固定导航
- 顶部状态栏
- 主内容区域
- 支持桌面端优先布局

### 3.2 全局状态

前端至少需要维护以下全局状态：

- 当前论文上下文
- 当前版本信息
- 最近任务状态
- 当前用户默认模型配置
- 全局通知与错误提示

### 3.3 公共交互要求

- 所有长耗时操作都要有进度反馈
- 所有失败状态都要有可理解的错误文案
- 所有高风险内容都要可点击定位
- 所有关键修改都要支持回退或取消

## 4. 页面级需求

## 4.1 工作台首页

### 功能目标

- 作为用户进入系统后的总入口
- 展示论文总览、最近论文、快捷操作、当前活跃任务

### 页面模块

- 欢迎区
- 数据统计卡
- 最近论文列表
- 快捷操作区
- 当前活跃任务卡
- 历史任务表
- 首页空状态

### 核心交互

- 点击上传新论文，打开上传弹窗
- 点击进入模型配置，跳转模型配置页
- 点击查看详情，进入论文详情页
- 点击继续改写，进入改写对比台
- 点击历史任务，进入对应结果页

### 页面状态

- 默认态
- 空状态
- 有活跃任务状态
- 加载中状态
- 接口失败状态

## 4.2 论文详情页

### 功能目标

- 展示文档解析后的章节和段落结构
- 允许用户选择排除封面、目录、参考文献、附录等区域

### 页面模块

- 文件信息区
- 章节目录区
- 正文预览区
- 排除范围设置区
- 版本信息区

### 核心交互

- 保存排除范围
- 重新解析
- 开始检测

### 页面状态

- 解析中
- 解析成功待确认
- 解析失败

## 4.3 检测报告页

### 功能目标

- 展示全文风险概览和段落风险定位
- 帮助用户从高风险段快速进入改写流程

### 页面模块

- 顶部分数摘要
- 正文高亮阅读区
- 风险详情侧栏
- 风险筛选区
- 去改写按钮

### 核心交互

- 点击高亮段落查看详情
- 切换高风险段
- 一键加入改写
- 从报告页跳转改写页

### 页面状态

- 检测中
- 检测完成
- 无高风险段
- 检测失败

## 4.4 改写对比台

### 功能目标

- 展示原文和改写结果
- 让用户逐段完成采纳、回退和再次改写

### 页面模块

- 顶部论文信息区
- 改写策略区
- 原文区
- 改写区
- 差异高亮区
- 底部操作区
- 整体处理进度区

### 核心交互

- 采纳建议
- 换一批
- 手动微调
- 回退到原文
- 复检当前段

### 页面状态

- 未生成改写
- 单候选结果
- 多候选结果
- 已采纳
- 已回退
- 改写中

## 4.5 模型配置页

### 功能目标

- 管理本地模型和云端模型配置
- 即时测试模型效果

### 页面模块

- 模型列表区
- 基础参数区
- 资源消耗估算区
- 即时测试区
- 保存按钮区

### 核心交互

- 新增模型配置
- 编辑模型配置
- 删除模型配置
- 测试连接
- 设为默认改写模型
- 设为默认检测模型

### 页面状态

- 未配置
- 已配置未测试
- 测试成功
- 测试失败

## 4.6 上传弹窗

### 功能目标

- 提供统一文件上传入口
- 展示上传中与解析中的进度

### 页面模块

- 拖拽上传区
- 文件格式说明
- 当前进度条
- 上传状态说明
- 取消按钮

### 页面状态

- 初始待上传
- 上传中
- 上传完成解析中
- 上传失败

## 5. 前端组件需求

建议优先沉淀以下组件：

- `AppSidebar`
- `TopBar`
- `StatCard`
- `PaperCard`
- `TaskTable`
- `RiskTag`
- `RiskScoreCard`
- `ParagraphHighlightBlock`
- `RewriteComparePane`
- `ModelConfigForm`
- `UploadProgressDialog`
- `EmptyStateCard`
- `ErrorBanner`

## 6. 前端状态管理建议

建议区分以下几类状态：

- 页面级状态：当前页面数据和 loading
- 文档级状态：当前论文、当前版本、当前段落
- 任务级状态：检测、改写、复检、导出进度
- 配置级状态：模型配置与默认模型

## 7. 前端接口使用要求

- 上传、检测、改写、复检、导出统一按异步任务处理
- 前端轮询间隔建议为 `2~5 秒`
- 所有任务型接口都要支持“处理中 / 已完成 / 失败”
- 所有列表接口都应能兼容空数组返回

## 8. 前端验收标准

- 用户可从首页完成一次完整闭环
- 检测报告页可准确展示段落级风险
- 改写页可完成采纳、回退和再次改写
- 模型配置页可完成保存和测试
- 上传弹窗可展示 0~100% 的过程状态
- 首页空状态清晰表达下一步动作

## 9. 前端默认假设

- 一期默认桌面端优先，不要求移动端完整适配
- 一期默认单用户，不接入复杂登录体系
- 一期默认接口均为 REST 风格

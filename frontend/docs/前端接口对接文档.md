# 前端接口对接文档

- 文档版本：`v1.0`
- 编写日期：`2026-03-21`
- 适用对象：`前端开发 / 联调负责人`

## 1. 文档目的

本文档用于说明前端页面应该调用哪些接口、如何组织状态、如何处理任务轮询和错误反馈。

## 2. 页面与接口映射

| 页面 | 主要接口 |
| --- | --- |
| 工作台首页 | `GET /api/dashboard/overview` `GET /api/dashboard/recent-documents` `GET /api/dashboard/active-task` `GET /api/dashboard/task-history` |
| 上传弹窗 | `POST /api/documents/upload` |
| 论文详情页 | `GET /api/documents/{docId}` `POST /api/documents/{docId}/parse` `PATCH /api/documents/{docId}/exclusions` |
| 检测报告页 | `GET /api/detection-tasks/{taskId}` `GET /api/documents/{docId}/detection-summary` |
| 改写对比台 | `POST /api/rewrite-tasks` `GET /api/rewrite-tasks/{taskId}` `POST /api/rewrite-candidates/{candidateId}/accept` `POST /api/rewrite-candidates/{candidateId}/rollback` |
| 模型配置页 | `GET /api/model-providers` `POST /api/model-providers` `PUT /api/model-providers/{providerId}` `DELETE /api/model-providers/{providerId}` `POST /api/model-providers/{providerId}/test` |
| 导出弹窗 | `POST /api/exports` `GET /api/exports/{exportId}` |

## 3. 前端推荐类型定义

```ts
export interface ApiResponse<T> {
  code: number;
  message: string;
  data: T;
  requestId: string;
}

export interface ModelProvider {
  providerId: string;
  providerType: "local" | "cloud";
  name: string;
  baseUrl: string;
  apiKey: string;
  model: string;
  capabilities: Array<"rewrite" | "detect">;
  timeoutMs: number;
  concurrency: number;
  contextWindow: number;
  status: "unknown" | "ok" | "error";
}

export interface SegmentResult {
  segmentId: string;
  originalText: string;
  riskScore: number;
  riskType: "plagiarism" | "aigc" | "mixed";
  evidence: {
    reason: string;
  };
  suggestedAction: string;
}

export interface RewriteCandidate {
  candidateId: string;
  segmentId: string;
  rewrittenText: string;
  explanation: string;
  beforeScore: number;
  afterScore: number;
  accepted: boolean;
}
```

## 4. 轮询策略

以下任务前端需要轮询：

- 文档解析
- 检测
- 改写
- 复检
- 导出

建议规则：

- 初始轮询间隔：`2 秒`
- 任务运行超过 30 秒后改为：`5 秒`
- 接口返回 `done` 或 `failed` 时停止轮询

## 5. 页面数据加载建议

## 5.1 工作台首页

进入页面后并行加载：

- 总览数据
- 最近论文
- 当前活跃任务
- 历史任务

## 5.2 检测报告页

优先加载：

- 检测结果总览
- 段落结果列表

需要支持局部更新：

- 高亮段详情
- 已处理状态

## 5.3 改写对比台

优先加载：

- 当前段落原文
- 当前改写候选
- 当前处理进度

支持局部刷新：

- 重新生成当前段
- 切换候选版本
- 采纳后局部刷新

## 6. 上传与导出处理

## 6.1 上传

- 使用 `multipart/form-data`
- 前端需展示上传进度
- 上传完成后显示“解析中”状态

## 6.2 导出

- 导出采用任务模式
- 任务完成后获取下载地址
- 前端需提示用户下载成功或失败

## 7. 错误处理要求

前端需统一处理以下错误：

- 参数错误
- 文件格式不支持
- 模型连接失败
- 模型调用失败
- 文档不存在
- 任务不存在
- 导出失败

建议前端错误提示文案直接来自后端 `message`，同时保留 `requestId` 便于排查。

## 8. Mock 数据建议

联调前建议优先制作以下 mock：

- 工作台首页 mock
- 检测报告页 mock
- 改写页 mock
- 模型配置页 mock
- 上传进度 mock

## 9. 联调顺序建议

1. 先联调模型配置页
2. 再联调上传与论文详情页
3. 再联调工作台首页
4. 再联调检测报告页
5. 最后联调改写、复检和导出

## 10. 一期默认假设

- 一期默认不做复杂权限拦截
- 一期默认页面数据可以一次拉取，不强制做细粒度拆分接口
- 一期默认前端使用 REST API，不接 WebSocket

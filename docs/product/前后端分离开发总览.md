# 论文降重工具前后端分离开发总览

- 文档版本：`v1.0`
- 编写日期：`2026-03-21`
- 适用对象：`产品 / 前端 / 后端`

## 1. 文档目的

本文档用于将现有产品需求、Figma 页面和接口设计整理为适合前后端分离开发的交付材料。

建议后续按以下方式拆分到不同目录：

- `frontend/docs/前端需求文档.md`
- `frontend/docs/前端接口对接文档.md`
- `backend/docs/后端需求文档.md`
- `backend/docs/后端接口文档.md`

## 2. 一期产品范围

一期聚焦以下核心闭环：

`上传论文` → `正文解析` → `风险检测` → `高风险段定位` → `段落改写` → `对比采纳` → `复检` → `导出报告`

对应核心模块：

- 工作台首页
- 论文详情页
- 检测报告页
- 改写对比台
- 模型配置页
- 上传弹窗
- 空状态 / 错误状态

## 3. 前后端职责边界

### 3.1 前端职责

- 页面渲染与交互实现
- 文件上传入口与上传进度展示
- 任务状态轮询与结果展示
- 检测报告、改写对比、模型配置等页面交互
- 差异高亮、段落选择、批量操作等前端体验逻辑

### 3.2 后端职责

- 文件接收、解析、存储
- 文档结构化与段落切分
- 模型配置持久化与测试
- 检测任务执行
- 改写任务执行
- 复检任务执行
- 导出任务生成
- 统一 API 返回和错误处理

## 4. 分离开发的统一约束

- 前端只通过 HTTP API 与后端通信，不直接访问模型服务或本地文件系统。
- 后端不承载页面逻辑，只提供数据、任务与状态接口。
- 长耗时任务统一走异步任务模式。
- 所有页面状态应能通过接口驱动，包括空状态、处理中、失败、已完成。
- 一期默认单用户，不做复杂权限系统，但接口保留 `Authorization` 扩展位。

## 5. 推荐开发顺序

1. 后端先完成文档、任务、模型配置的基础数据结构。
2. 前端先完成工作台首页、模型配置页和上传弹窗的静态页面。
3. 后端完成文档上传、解析、模型测试接口。
4. 前端对接上传与模型配置流程。
5. 后端完成检测、改写、复检、导出接口。
6. 前端完成检测报告页和改写对比台联调。

## 7. 一期默认假设

- 产品为个人论文助手，不做机构多用户协作。
- Web 端优先，桌面端暂不纳入开发范围。
- 中文论文优先，兼容少量英文摘要和英文段落。
- 检测能力以内置检测为主，第三方平台只预留接口扩展位。
- 模型统一通过适配层接入，本地模型与云端模型使用统一抽象。

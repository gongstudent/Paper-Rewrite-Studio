# 论文降重助手测试案例文档

- 文档版本：`v1.0`
- 编写日期：`2026-03-21`
- 适用对象：`开发自测 / 手工测试 / 验收测试`
- 测试目标：`覆盖当前降重助手的全部主要功能`

## 1. 测试目的

本文档用于系统性验证论文降重助手当前版本的主要功能是否正常，包括：

- 页面可访问
- 前后端接口可用
- 关键按钮可点击并触发正确行为
- 上传、解析、检测、改写、复检、导出闭环可跑通
- 边界情况和错误提示可理解

## 2. 测试环境

### 2.1 服务地址

- 前端：`http://localhost:5173`
- 后端：`http://localhost:3100`
- Swagger：`http://localhost:3100/api/docs`

### 2.2 建议测试文件

建议准备这些测试文件：

- `txt`：短论文样例
- `docx`：中文论文样例
- `pdf`：可提取文本的样例
- `doc`：旧版 Word 文档样例

### 2.3 当前已知约束

- `.doc` 当前不支持自动解析，应提示先转换为 `.docx / pdf / txt`
- 本地为单用户模式
- 检测和改写为任务式处理

## 3. 功能测试案例

## 3.1 工作台首页

### TC-W01 首页正常加载

- 操作步骤：
  1. 打开工作台首页
- 预期结果：
  1. 左侧导航、顶部栏、统计卡、近期论文、快捷操作区域均正常显示
  2. 无阻塞性 404/500 错误

### TC-W02 首页空状态

- 前置条件：系统中无论文
- 操作步骤：
  1. 打开工作台首页
- 预期结果：
  1. 显示上传第一篇论文的空状态
  2. 主按钮可打开上传弹窗

### TC-W03 近期论文卡片展示

- 前置条件：系统中已有至少 2 篇论文
- 操作步骤：
  1. 打开工作台首页
- 预期结果：
  1. 显示论文标题、更新时间、版本、状态标签
  2. 文件名和风险标签正常显示

### TC-W04 工作台快捷入口可用

- 前置条件：已有论文数据
- 操作步骤：
  1. 点击 `上传论文`
  2. 点击 `开始检测`
  3. 点击 `高风险改写`
  4. 点击 `模型配置`
- 预期结果：
  1. 上传按钮打开上传弹窗
  2. 开始检测跳转到论文详情页
  3. 高风险改写跳转到正确下一步（有报告则进报告，无报告则进详情）
  4. 模型配置跳转成功

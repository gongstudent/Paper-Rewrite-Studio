# 论文降重工具需求文档

- 文档版本：`v1.0`
- 编写日期：`2026-03-20`
- 产品阶段：`一期需求定义`
- 产品定位：`个人论文助手`
- 交付形态：`Web 端优先`
- 语言范围：`中文论文优先，兼容英文摘要与少量英文段落`

## 1. 产品概述

论文降重工具是一款面向学生、研究者与学术写作者的论文辅助产品，目标是帮助用户在论文撰写与提交前完成以下闭环：

1. 上传论文并解析正文。
2. 对全文进行重复风险检测与 AIGC 风险检测。
3. 定位高风险段落并进行按段改写。
4. 查看原文与改写结果对比，人工选择采纳、回退或继续改写。
5. 复检并导出最终稿或差异报告。

本产品不是通用写作助手，而是围绕论文场景构建的“检测 + 改写 + 复检 + 对比采纳”工具。产品强调可解释、可控制、可回退，不承诺“必过”或“保证降至某个阈值”，而是帮助用户更高效地完成论文质量优化与人工复核。

## 2. 背景与目标

### 2.1 背景

随着生成式 AI 在论文写作中的广泛使用，用户面临两类核心风险：

- 论文内容可能存在较高的文本重复率风险。
- 论文内容可能被外部系统识别出较高的 AIGC 风险。

与此同时，用户通常还面临如下问题：

- 无法快速定位真正需要处理的高风险段落。
- 使用通用改写工具后，学术表达被破坏，引用、术语、数字和结构易失真。
- 无法直观看到“改写前后到底变了什么”。
- 本地模型与云端模型切换麻烦，缺少统一配置入口。

### 2.2 产品目标

- 为用户提供论文上传、检测、改写、复检、导出的一站式体验。
- 支持本地模型与云端模型配置，降低用户接入模型的技术门槛。
- 以段落为单位输出风险解释和改写结果，提升可控性。
- 帮助用户保留论文的学术风格、术语准确性与引用结构。
- 形成“检测-改写-复检”的可追踪闭环。

### 2.3 非目标

- 不提供机构版送检、审批、班级管理、审计留痕等能力。
- 不承诺与任何学校、期刊或商业检测平台的最终结果完全一致。
- 不提供自动引用修复、文献管理、模板排版等非核心一期能力。
- 不以“绕过检测”为产品卖点，不提供违规承诺。

## 3. 目标用户与核心场景

### 3.1 目标用户

- 本科生、硕士生、博士生
- 需要提交课程论文、毕业论文、期刊论文的研究者
- 需要在正式查重或送审前进行自检和修改的用户

### 3.2 核心场景

#### 场景 A：投稿前的全文自检

用户上传论文初稿，系统自动解析正文并做检测，输出全文风险总览和高风险段落列表。用户优先处理高风险段落，再进行复检。

#### 场景 B：仅针对高风险片段进行改写

用户不希望全文自动改写，而是只选择标红或高风险段落逐段处理，并对每次改写结果进行人工选择。

#### 场景 C：切换不同模型比较改写效果

用户可能同时拥有本地模型与云端模型，希望在同一产品中配置多个模型，测试不同模型在“降重复率”与“降低 AIGC 痕迹”上的表现。

#### 场景 D：保守改写，避免破坏学术内容

用户需要保留专业术语、公式编号、数据指标、文献引用及学术结构，不希望系统把论文改成口语化、营销化或偏离原意的文本。

## 4. 产品原则

- 可解释：所有检测结果都应尽可能展示段落级风险、原因说明和建议动作。
- 可控：改写以段落为最小单位，用户可逐段采纳、回退、重试。
- 可复检：每次改写后都可以重新检测，看到风险变化。
- 可扩展：模型适配层支持本地与云端，后续可扩展第三方检测导入。
- 合规优先：产品强调学术辅助，不鼓励虚假原创或规避规则。

## 5. 一期范围

### 5.1 核心闭环

一期必须支持以下主流程：

`上传论文` → `文本解析与分段` → `内置检测` → `高风险片段定位` → `按段改写` → `原文/改写对比` → `人工采纳或回退` → `复检` → `导出结果/报告`

### 5.2 一期模块

- 论文工作台
- 模型配置中心
- 检测中心
- 改写工作台
- 对比与采纳
- 报告与历史记录

## 6. 功能需求

## 6.1 论文工作台

### 6.1.1 功能说明

论文工作台用于承载用户的论文上传、解析、分段、版本管理与任务入口。

### 6.1.2 功能点

- 支持上传 `doc`、`docx`、`pdf`、`txt` 文件。
- 支持创建新的论文项目并记录标题、语言、来源文件、更新时间。
- 支持正文解析与结构识别，初步拆分为章节、段落、句子。
- 支持用户选择是否排除以下内容：
  - 封面
  - 目录
  - 参考文献
  - 附录
- 支持识别并保留论文中的基本结构元素：
  - 标题
  - 一级/二级标题
  - 段落
  - 引用块
  - 表格/公式编号文本
- 支持保存论文版本快照，供后续回退或对比。

### 6.1.3 交互要求

- 上传完成后进入解析态，展示解析进度。
- 解析完成后提供“分段预览”页面，用户可确认哪些区域纳入检测。
- 若 PDF 解析失败或结构识别不完整，应提示用户切换文件格式或手动调整排除范围。

## 6.2 模型配置中心

### 6.2.1 目标

统一配置本地模型与云端模型，使用户无需理解底层调用差异，即可完成连通性测试、参数保存与调用。

### 6.2.2 支持的模型类型

#### 本地模型

- `Ollama`
- `OpenAI-compatible 本地网关`

#### 云端模型

- `OpenAI-compatible 抽象接入`
- 一期预置配置位：
  - `OpenAI`
  - `DeepSeek`
  - `阿里云百炼`
  - `Anthropic`
  - `Gemini`

### 6.2.3 配置字段

每个模型提供方至少包含以下配置：

- 提供方名称
- 提供方类型：`local` / `cloud`
- `Base URL`
- `API Key / Token`
- `Model`
- 请求超时
- 最大并发数
- 上下文长度
- 能力标签：`rewrite` / `detect`
- 连通性状态

### 6.2.4 功能点

- 新建模型配置
- 编辑模型配置
- 删除模型配置
- 设置默认改写模型
- 设置默认检测模型
- 测试连接
- 查看最近一次调用状态
- 为不同任务绑定不同模型

### 6.2.5 异常处理

- 当 Token 缺失或无效时，应明确提示鉴权失败。
- 当 Base URL 无法访问时，应提示网络或服务地址异常。
- 当模型不存在时，应提示用户检查模型名称。
- 当本地模型服务未启动时，应提供明确的修复建议。

## 6.3 检测中心

### 6.3.1 检测能力

一期定义两类内置检测能力：

#### 重复风险检测

- 输出全文风险分
- 输出段落风险分
- 输出疑似重复片段
- 输出高风险句段排序
- 支持按章节、段落、风险等级筛选

#### AIGC 风险检测

- 输出全文风险分
- 输出段落级风险热区
- 输出模型判定说明
- 支持复检结果前后对比
- 支持按高风险段落聚焦改写

### 6.3.2 检测结果要求

检测结果页需要至少展示以下信息：

- 检测任务状态
- 全文总览
- 高风险段落数量
- 风险分布图
- 高风险段落列表
- 每段的风险类型
- 风险说明
- 建议动作

### 6.3.3 筛选与跳转

- 支持只看高风险段
- 支持只看未处理段
- 支持从检测报告一键跳转到改写工作台
- 支持按段落批量加入改写队列

### 6.3.4 一期实现策略

- 一期以内置检测为主。
- 外部检测平台只做接口预留，不承诺直接打通。
- 系统需保留导入第三方检测报告的扩展位，支持后续兼容知网、维普、万方、Turnitin、Copyleaks 等结果结构。

## 6.4 改写工作台

### 6.4.1 目标

围绕论文文本的高风险段落进行场景化改写，而不是提供面向所有写作类型的泛化改写。

### 6.4.2 改写粒度

- 逐段改写
- 批量改写
- 针对选中句段的局部改写

### 6.4.3 改写策略

一期至少支持以下策略：

- 降重复率
- 降低 AIGC 痕迹
- 学术表达增强
- 保留术语/引用/数字

### 6.4.4 提示控制能力

系统应支持以下控制项：

- 保守改写 / 平衡改写 / 强改写
- 是否保留专有名词
- 是否保留参考文献标记
- 是否保留数字和单位
- 是否优先学术书面表达
- 是否避免口语化

### 6.4.5 输出内容

每一次改写至少输出：

- 改写稿
- 改写说明
- 使用的模型信息
- 使用的改写策略
- 改写前风险分
- 改写后风险分

### 6.4.6 交互要求

- 用户可在检测报告中勾选多个高风险段后发起批量改写。
- 对同一段落支持多次改写并保留候选结果。
- 改写完成后默认进入对比视图。

## 6.5 对比与采纳

### 6.5.1 核心目标

“对比与采纳”是一期开启用户信任的核心模块，必须做到直观、稳定、可逆。

### 6.5.2 视图要求

每段展示以下信息：

- 原文
- 改写稿
- 差异高亮
- 改写说明
- 检测前分值
- 改写后分值
- 当前状态

### 6.5.3 用户动作

- 采纳当前改写
- 回退到原文
- 重新改写
- 切换候选版本
- 标记为稍后处理
- 一键采纳当前页全部已选结果

### 6.5.4 全文视角

需要支持全文模式，帮助用户快速查看：

- 哪些段已替换
- 哪些段仍高风险
- 哪些段已经复检
- 哪些段是最新版本

## 6.6 报告与历史记录

### 6.6.1 功能点

- 查看检测任务历史
- 查看改写任务历史
- 查看版本快照
- 查看复检结果
- 导出最终稿
- 导出差异报告

### 6.6.2 导出类型

- 仅最终稿
- 带差异标记稿

### 6.6.3 历史可追踪性

系统至少保留以下历史信息：

- 创建时间
- 使用的模型
- 使用的任务类型
- 风险分变化
- 是否被用户采纳

## 8. 核心流程

### 8.1 主流程

1. 用户上传论文文件。
2. 系统解析正文并生成段落结构。
3. 用户确认排除范围。
4. 系统执行重复风险检测与 AIGC 风险检测。
5. 系统生成检测报告并标出高风险段。
6. 用户选择高风险段进入改写工作台。
7. 系统基于指定模型和策略生成改写候选。
8. 用户查看原文与改写结果差异，决定采纳、回退或再次改写。
9. 系统对采纳后的内容执行复检。
10. 用户导出最终稿或差异报告。

### 8.2 失败与中断流程

- 解析失败：提示用户切换格式或重新上传。
- 模型调用失败：保留当前任务状态并提供重试入口。
- 改写中断：已完成段落的结果不丢失，未完成段落可继续执行。
- 复检失败：保留改写结果，但标记“待复检”状态。

## 10. 外部接口与系统边界

### 10.1 模型调用适配接口

用于统一调用本地模型与云端模型，屏蔽不同服务商之间的协议差异。

#### 一期要求

- 支持 `OpenAI-compatible` 风格的文本生成接口适配。
- 支持将同一段改写任务切换到不同提供方执行。
- 支持返回标准化错误码和错误文案。

### 10.2 第三方检测导入接口

用于后续接收外部查重或 AIGC 检测平台的结构化结果。

#### 一期要求

- 仅预留接口与数据结构，不作为必须交付能力。
- 导入能力未来可兼容：
  - 知网
  - 维普
  - 万方
  - Turnitin
  - Copyleaks

## 11. 非功能需求

### 11.1 性能

- 单篇中等篇幅论文应支持在可接受时间内完成解析与检测。
- 批量改写时需要提供明确的进度反馈。
- 对长文本段落应提供自动拆分与重试机制。

### 11.2 可用性

- 非技术用户应能独立完成模型配置与论文处理。
- 核心操作路径不超过 3 次页面跳转。
- 重要按钮文案清晰，避免术语过多。

### 11.3 稳定性

- 模型调用失败不应导致任务数据丢失。
- 解析、检测、改写、复检任务需要保留任务状态。

### 11.4 安全与隐私

- 用户上传文件需要有明确的存储与删除策略。
- 涉及 API Key 的配置必须脱敏展示。
- 对云端模型调用要明确告知“文本将发送到外部服务”。

## 12. 合规约束

产品文案、能力定义与交互提示中需要明确以下约束：

- 不承诺查重平台最终结果。
- 不承诺规避学校或期刊检测。
- 不承诺将结果降低至某个确定阈值。
- 用户应保留真实引用、参考文献与事实准确性。
- AI 改写结果仅作为辅助建议，最终内容需用户自行审核与承担责任。
- 对于可能改变原意、引用关系或数据表述的改写，应提醒用户进行人工校对。

## 13. 竞品调研补充

以下调研基于 `2026-03-20` 可访问的官方公开页面，作为需求补充，不作为广告或能力承诺依据。

### 13.1 国内工具侧重点

#### PaperPass

- 官方页面展示了 `AI 智能降重检测系统`，并列出 `报告降重增强版`、`全文降重增强版`、`报告降 AIGC 率`、`全文降 AIGC 率` 等服务形态。
- `报告降 AIGC 率` 页面公开说明支持基于报告与原文的流程处理，并提到兼容部分外部 AIGC 检测报告。
- 这类产品的价值在于“报告驱动处理”，说明用户很在意依据检测结果对高风险内容进行定向优化。

对本产品的启发：

- 一期必须突出“从检测报告直接发起改写”的链路。
- 必须支持针对高风险段定向处理，而不是只做全文通用改写。

#### 火龙果写作

- 官网与英文功能页显示其能力覆盖 `语法检查`、`改写`、`续写`、`翻译`，并支持桌面端与 Office/WPS/PDF 等插件形态。
- 这说明写作辅助工具的竞争优势之一是多终端与低门槛接入。

对本产品的启发：

- 一期虽然以 Web 端优先，但模型接入与任务操作要足够轻量。
- 后续应预留桌面端扩展能力，以便更好支持本地模型与本地文件工作流。

### 13.2 国外工具侧重点

#### QuillBot

- 官方提供 `Paraphraser` 与 `Plagiarism Checker`，查重页公开说明支持上传 `txt`、`pdf`、`docx`、`html` 等格式，并输出潜在来源与报告。
- 官方帮助中心明确表示，改写工具并不保证能改善查重分数，引用仍然必须正确保留。

对本产品的启发：

- 需求文档中必须写入合规提醒，不能把“改写”直接等同于“规避查重”。
- 文件上传与检测报告的体验需要足够直观。

#### Turnitin

- 官方产品页显示其 `Feedback Studio` 可结合 `Similarity Report` 与 `AI writing indicator` 使用，并能把 AI 相关结果整合到报告视图中。
- 页面还提到 AI 写作识别可用于查看总体比例，并支持对可疑区域做进一步调查。

对本产品的启发：

- 我们的检测报告页应采用“总览 + 段落定位 + 风险解释”的结构。
- AIGC 风险展示不应只给一个总分，还应给出段落热区与解释。

#### Copyleaks

- 官网强调其 `AI Detector`、`Plagiarism Checker` 与 `API Integration / LMS Integration` 能力，适合平台级集成。
- API 页说明其可处理文本、文件、OCR 场景，并面向系统集成提供扩展能力。

对本产品的启发：

- 我们要在一期需求中把外部接口抽象出来，避免未来接第三方平台时推翻现有模型。
- 第三方检测导入接口与标准化任务对象应从一期就确定。

### 13.3 本产品差异化定位

与现有工具相比，本产品的一期差异化重点如下：

- 聚焦论文场景，而非通用写作场景。
- 聚焦段落级“检测-改写-采纳-复检”闭环，而非一次性全文改写。
- 聚焦本地模型与云端模型统一配置，而非单一厂商能力绑定。
- 聚焦对比、回退和人工决策，而不是黑盒式自动改写。

## 14. 验收标准

### 14.1 功能验收场景

- 上传 `doc`、`docx`、`pdf`、`txt` 后可正确解析正文，并允许用户排除封面、目录、参考文献、附录。
- 本地模型与云端模型都可完成连通性测试、失败提示与默认参数保存。
- 全文检测能输出总体风险、段落级风险与高风险定位。
- 用户可只改写高风险段，不影响其他段落。
- 改写后能查看原文、改写、差异和分数变化，并支持采纳、回退、再次改写。
- 复检后能显示前后对比，形成完整闭环。
- 导出结果时可选择仅最终稿或带差异标记稿。
- 当模型调用失败、额度不足、超时、段落过长时，系统能给出可操作提示。
- 当文本包含引用、公式、表格编号、专有名词时，改写策略应尽量保留学术结构。

### 14.2 默认验收指标

- 用户能在单次会话内完成一次完整闭环。
- 高风险段定位准确且具备解释性。
- 改写结果可被人工逐段控制。
- 模型配置无需技术人员介入即可完成。

## 15. 版本规划建议

### 15.1 一期必须交付

- 上传与解析
- 模型配置中心
- 内置检测
- 段落级改写
- 对比采纳
- 复检
- 导出与历史记录

### 15.2 二期可扩展

- 第三方检测报告导入
- 桌面端应用
- 更多文件格式解析优化
- 模板化改写策略
- 协作与分享能力

### 15.3 暂不纳入当前范围

- 自动引用修复
- 文献管理
- 学校模板排版
- 机构级私有化部署
- 机构账号与审批流程

## 16. 开发落地建议

为了降低实现风险，建议后续设计与开发优先按以下顺序推进：

1. 先完成文档解析、段落结构化和版本模型。
2. 再完成模型配置中心和统一调用适配层。
3. 随后实现检测报告页与高风险段筛选链路。
4. 最后实现改写工作台、对比采纳、复检与导出。

这样可以优先打通最关键的数据流：`文档` → `段落` → `检测结果` → `改写候选` → `采纳版本` → `复检结果`。

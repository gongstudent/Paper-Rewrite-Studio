## 2. 检测报告详情页

### 2.1 页面定位

该页面用于承接论文检测后的结果展示，核心目的是回答三个问题：

- 风险有多高
- 风险在哪些段落
- 用户下一步应该处理什么

### 2.2 当前页面结构

页面采用 `左侧导航 + 顶部状态栏 + 中间正文 + 右侧报告摘要` 布局。

主要区域包括：

- 左侧导航栏
- 顶部摘要栏
- 中间论文正文区
- 右侧报告概览卡
- 右侧段落风险详情卡
- 底部行动按钮

### 2.3 页面内容解读

#### 顶部摘要栏

顶部当前直接显示：

- 查重率
- AI 指数
- 检测耗时
- PDF 标记
- `去改写` 按钮

这块很好，说明用户一进入页面就能看懂结果总况。

#### 正文区

正文区展示了实际论文内容，并在文中直接用彩色风险块标出高风险段。

当前风险表达方式包括：

- 红色高亮块：偏重复率风险
- 紫色高亮块：偏 AIGC 风险

这是一种很好的“正文即报告”的表现方式，比单独列表更直观。

#### 右侧概览区

右侧顶部有 `报告概览` 卡，显示：

- 查重率 15%
- AI 指数 22%
- 日期
- 时间

#### 右侧风险详情区

右侧下方是 `段落风险详情`，分别解释：

- 查重风险点
- AI 生成风险点

每个风险点都包含：

- 对比来源或判断理由
- 风险百分比
- 补充说明

底部有明显的 `一键跳转去改写` 按钮。

### 2.4 对应接口建议

该页面主要需要：

- `GET /api/detection-tasks/{taskId}`
- `GET /api/documents/{docId}/detection-summary`
- `GET /api/documents/{docId}`

建议核心返回字段包括：

- `plagiarismScore`
- `aigcScore`
- `elapsedSeconds`
- `segmentResults`
- `riskType`
- `evidence`
- `suggestedAction`

## 3. 改写对比台

### 3.1 页面定位

该页面用于承接高风险段落的改写、比较与采纳，是整套产品最关键的操作页面之一。

### 3.2 当前页面结构

页面采用 `左侧导航 + 顶部上下文栏 + 双栏对比主体 + 底部处理进度` 结构。

主要区域包括：

- 左侧导航
- 顶部论文信息栏
- 顶部改写策略栏
- 左侧原文风险段落
- 右侧 AI 改写建议
- 底部操作区
- 页面底部整体处理进度提示

### 3.3 页面内容解读

#### 顶部信息栏

当前展示：

- 页面标题：`学术论文对比改写`
- 论文名称
- 版本号
- `版本保存`
- `开始改写`

#### 策略栏

策略区展示了：

- 改写强度：`均衡化`
- 学术语气：`学术风`
- `重新生成`

这说明页面已经具备“参数可控”的设计思路，很适合论文场景。

#### 左侧原文区

左侧标题为：

- `原文风险段落`

并展示：

- 当前风险值 82%
- 段落编号
- 原文内容

原文中高风险表达被划线或强调，便于用户理解问题来源。

#### 右侧改写区

右侧标题为：

- `AI 改写建议`

并展示：

- 预计风险分降低值
- 改写后的文本
- 不同颜色高亮
- 解释性标注，如 `卷积神经网络`、`诸如改进型` 等

#### 底部操作区

当前包含：

- `采纳建议`
- `换一批`
- `手动微调`

页面最底部还有整体进度：

- `当前已处理进度：4/12 段`
- `剩余风险点：8 个`
- `学术诚信保护已开启`

### 3.4 对应接口建议

该页面主要需要：

- `POST /api/rewrite-tasks`
- `GET /api/rewrite-tasks/{taskId}`
- `POST /api/rewrite-candidates/{candidateId}/accept`
- `POST /api/rewrite-candidates/{candidateId}/rollback`
- `POST /api/rewrite-tasks/{taskId}/retry`

建议核心返回字段包括：

- `segmentId`
- `originalText`
- `riskScore`
- `rewrittenText`
- `beforeScore`
- `afterScore`
- `explanation`
- `accepted`

## 4. 模型配置页

### 4.1 页面定位

该页面用于配置模型参数与即时测试，是本地模型和云端模型统一接入的核心页面。

### 4.2 当前页面结构

页面采用 `左侧系统导航 + 中间参数配置 + 右侧即时测试区` 结构。

主要区域包括：

- 顶部一级导航
- 左侧功能导航
- 中间基础参数卡
- 左下资源消耗卡
- 右侧即时预览测试区

### 4.3 页面内容解读

#### 顶部区域

当前顶部主导航包括：

- 总览
- 模型配置
- 知识库管理

右上角有：

- 消息图标
- 设置图标
- 用户头像

#### 基础参数区

左侧基础参数卡包括：

- 核心模型选择
- API Key
- 推理速度与质量平衡滑杆
- 语义一致性校验开关
- 实时引用追踪开关

这比普通“只填 Base URL 和 Model”的配置页更有产品感。

#### 资源消耗估算区

左下角增加了资源消耗预估，这是个非常好的点，因为它能帮助用户理解不同模型和不同参数的成本差异。

#### 即时预览测试区

右侧测试区包含：

- 测试文本输入框
- `开始即时测试` 按钮
- 模型输出结果区
- 当前推理引擎信息
- 延迟与 token 数据

这说明你不是只做“配置页”，而是在做“可验证的配置中心”，方向是对的。

### 4.4 对应接口建议

该页面主要需要：

- `GET /api/model-providers`
- `POST /api/model-providers`
- `PUT /api/model-providers/{providerId}`
- `DELETE /api/model-providers/{providerId}`
- `POST /api/model-providers/{providerId}/test`
- `POST /api/model-providers/{providerId}/set-default`

建议核心字段包括：

- `providerType`
- `name`
- `baseUrl`
- `apiKey`
- `model`
- `timeoutMs`
- `concurrency`
- `contextWindow`
- `status`

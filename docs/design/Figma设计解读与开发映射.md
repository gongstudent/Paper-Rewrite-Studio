## 1. 当前 Figma 设计概览

当前 Figma 节点对应的是产品的 `工作台首页`，整体为桌面端布局，画板尺寸约为 `1280 x 1207`。

页面结构分为三大区域：

- 左侧固定侧边导航
- 顶部共享导航栏
- 中间主工作区

从整体视觉来看，这版设计已经具备比较明确的“学术工作台”方向，风格偏理性、轻量、整洁，适合作为一期主视觉基调。

## 3. 这版设计做得好的地方

### 3.1 信息层级清晰

页面首先告诉用户“这里是什么”，然后给出“我现在该做什么”，再给出“我最近做过什么”。逻辑顺序是对的。

### 3.2 首页价值表达明确

上传、检测、改写、复检、导出这条主链路已经在首页被显式表达出来了，用户进入后不会迷失。

### 3.3 视觉风格较稳

整体留白充足，卡片布局清楚，颜色控制克制，比较符合论文类产品调性。

### 3.4 右侧快捷操作非常有用

这块能明显提升新用户启动效率，也适合后续放到演示稿和落地页中作为核心功能说明。

## 4. 建议优化的点

## 4.1 品牌名称与产品名称不一致

左侧品牌区当前英文品牌名和页面主体中文产品名不一致，会让产品显得像模板改稿，而不是完整产品。

建议统一：

- 中文版统一为 `论文降重工具` 或 `论文工作台`
- 英文辅助名作为副标题即可

## 4.2 当前“高风险”表达还不够细

首页中已经有风险提示，但还没有明显区分：

- 重复率风险
- AIGC 风险

建议在论文卡片和统计卡片中明确拆开两种风险，避免用户以为只有一种检测维度。

例如：

- `AIGC 风险：高`
- `重复率风险：警告`

或采用双色小标签并列展示。

## 4.3 最近论文卡片的信息还可以更完整

目前卡片已具备标题、版本和风险信息，但建议再加入：

- 最近检测时间
- 最近改写时间
- 当前处理状态

例如：

- `待复检`
- `已处理 4/12 段`
- `最近检测：2 小时前`

这样用户能更快判断下一步该点哪篇论文。

## 4.4 任务历史表格建议强化“任务类型差异”

现在底部表格已经有检测任务和改写任务两个 Tab，但表格字段风格仍较统一。

建议增加更明显的任务类型差异：

- 检测任务展示：`检测类型 / 风险摘要 / 状态`
- 改写任务展示：`改写策略 / 采纳数量 / 状态`

## 4.5 快捷操作区可以再增强主次关系

目前所有快捷按钮权重比较接近。建议突出最核心的两个：

- 上传论文
- 开始检测

其余操作作为次级入口。

这样更符合首页第一次进入时的用户路径。

## 4.6 浮动按钮要考虑和主流程是否重复

右下角当前有 `上传新初稿` 的浮动按钮。如果顶部欢迎区已经有主按钮，可能会出现动作重复。

建议二选一：

- 要么保留顶部主按钮，去掉悬浮按钮
- 要么保留悬浮按钮，但改成 `开始新任务`

## 6. 这个页面需要的接口

## 6.1 首页概览接口

### `GET /api/dashboard/overview`

返回：

```json
{
  "paperCount": 24,
  "highRiskSegments": 12,
  "todayDetectionTasks": 8,
  "todayRewriteAcceptRate": 86
}
```

## 6.2 最近论文列表接口

### `GET /api/dashboard/recent-documents`

返回：

```json
{
  "items": [
    {
      "docId": "doc_001",
      "title": "基于大语言模型的学术写作伦理研究",
      "updatedAt": "2026-03-20T10:20:00+08:00",
      "version": 4.2,
      "aigcRiskLevel": "high",
      "plagiarismRiskLevel": "warning",
      "status": "pending_recheck"
    }
  ]
}
```

## 6.3 当前活跃任务接口

### `GET /api/dashboard/active-task`

返回：

```json
{
  "taskId": "task_001",
  "taskName": "检测中：自动化伦理探讨.docx",
  "taskType": "detection",
  "progress": 78,
  "elapsedText": "剩余约 45 秒",
  "status": "running"
}
```

## 6.4 历史任务接口

### `GET /api/dashboard/task-history?tab=detection`

返回：

```json
{
  "items": [

      "taskId": "429",

      "startedAt": "10:45 AM",
      "analysisType": "mixed",
      "status": "done"
```

## 7. 页面与后续页面的跳转关系

当前页面中建议建立如下跳转：

- 点击 `上传新论文` → 上传弹窗 → 论文详情页
- 点击 `进入模型配置` → 模型配置页
- 点击论文卡片 `查看详情` → 论文详情页
- 点击论文卡片 `继续改写` → 改写对比台
- 点击快捷操作 `开始检测` → 检测报告页或检测任务发起弹窗
- 点击任务历史行 → 任务详情 / 对应结果页
